<task type="auto">
  <name>Task 2: Configure Railway deployment with Bree process</name>
  <files>railway.toml, src/scheduler/index.ts</files>
  <action>
    **railway.toml** — Create at project root. Railway uses this to determine how to run the app. We need TWO processes: the Next.js web server and the Bree scheduler (persistent process required — Vercel cannot run Bree).

    ```toml
    [build]
    builder = "NIXPACKS"

    [deploy]
    startCommand = "npm run start"
    healthcheckPath = "/"
    healthcheckTimeout = 30
    restartPolicyType = "ON_FAILURE"
    restartPolicyMaxRetries = 3

    [[services]]
    name = "web"
    startCommand = "npm run start"

    [[services]]
    name = "scheduler"
    startCommand = "node src/scheduler/index.js"
    ```

    Note: Railway's multi-service setup uses separate services in the dashboard. The `railway.toml` documents the intent. In practice, for a single Railway service deployment, `startCommand` runs only Next.js. The scheduler will run as a separate Railway service pointing to the same repo with `startCommand = "node src/scheduler/index.js"`.

    Add to **package.json** scripts:
    ```json
    "scheduler": "node src/scheduler/index.js"
    ```

    **src/scheduler/index.ts** — Create the Bree entry point (stub for now, Phase 4 adds jobs):
    ```typescript
    import Bree from 'bree';
    import path from 'path';

    const bree = new Bree({
      root: path.join(__dirname, 'jobs'),
      jobs: [
        // Phase 4 will add: { name: 'send-messages', interval: '1m' }
      ],
    });

    bree.start().then(() => {
      console.log('Bree scheduler started — no jobs configured yet (Phase 4)');
    });
    ```

    Create **src/scheduler/jobs/.gitkeep** so the jobs directory exists in git.

    Also add to **package.json**:
    ```json
    "build:scheduler": "tsc --project tsconfig.json --outDir dist"
    ```

    The scheduler runs as compiled JS in production (`node src/scheduler/index.js` after build). For now the stub is enough to confirm Railway can start the process.
